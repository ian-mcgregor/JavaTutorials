Type Wrappers, Auto-boxing and Unboxing



Primitive types include int, double, ect are not objects (on the heap). When variables of primitive types are in scope, memory is allocated on the stack for them. There are times when we want an object representation of primitive types. Each primitive type has a wrapper class associated with it. These classes have fields, constructors and methods that can be very useful. The type wrappers are: Double, Float, Long, Integer, Short, Byte, Character and Boolean. 

		int counter = 0;

		Integer counter2 = new Integer(counter);

AutoBoxing and Unboxing

A variable of a primitive wrapper type can be initialized with a primitive. This is called auto-boxing. When auto-boxing occurs, the system actually creates a new wrapper typed object automatically (line 2 below). Similarly we can get the primitive value stored in a wrapper typed object using a simple assignment statement (line 3 below). This is called unboxing. 

		int counter1 = 0;
		Integer counter2 = counter1;          // auto-boxing
		int counter3 = counter2;              // unboxing
